---
title: Local development
description: Run Classmoji on your machine and contribute code.
---

Minimal setup covers most contributions. Full setup is only needed if you're working on GitHub-integrated features like repository sync, webhooks, or background jobs.

## Prerequisites

- Node.js 22+
- [Docker](https://www.docker.com/) (if using the Docker database option)
- A GitHub account

## 1. Fork and clone the repo

Fork the repo on GitHub first, then clone your fork:

```bash
git clone https://github.com/YOUR_USERNAME/classmoji.git
cd classmoji
git remote add upstream https://github.com/classmoji/classmoji.git
npm install
```

Adding `upstream` lets you pull in future changes from the main repo with `git fetch upstream`.

## 2. Configure environment variables

```bash
cp .env.example .env
```

Your `.env` will look something like this to start:

```env
WEBAPP_URL="http://localhost:3000"
SLIDES_URL="http://localhost:6500"
PAGES_URL="http://localhost:7100"
BETTER_AUTH_BASE_URL={WEBAPP_URL}
BETTER_AUTH_SECRET="your-auth-secret"
DATABASE_URL="postgresql://classmoji:classmoji@localhost:5433/classmoji"
GITHUB_WEBHOOK_SECRET="your-webhook-secret"
```

The GitHub App credentials (`GITHUB_APP_ID`, `GITHUB_CLIENT_ID`, etc.) get added automatically when you complete the setup wizard in the next steps.

## 3. Database setup

Choose one option. Both work with the rest of the setup.

### Docker

No account needed, runs fully offline. Install [Docker](https://www.docker.com/) if you don't have it.

```bash
npm run db:start
```

The `DATABASE_URL` in `.env.example` already points to the Docker instance — no changes needed.

### Neon

Neon is a serverless PostgreSQL platform with a generous free tier. It's a great option if you can't run Docker or want to visualize your database — Neon provides a built-in UI where you can browse tables and run queries directly in the browser.

1. Create a free project at [neon.com](https://neon.com)
2. Copy the connection string into your `.env` as `DATABASE_URL`

---

### Apply migrations and seed

Regardless of which option you chose:

```bash
npm run db:deploy   # applies migrations
npm run db:seed     # seeds dev classroom and fake users
```

## 4. Run the app

```bash
npm run dev
```

Open `http://localhost:3000` — you'll be taken to a setup wizard. Click **Create GitHub App on Github**, confirm on GitHub, and you'll be redirected back with everything configured automatically.

After the wizard completes, these variables will be added to your `.env` automatically:

```env
GITHUB_APP_ID="your-app-id"
GITHUB_PRIVATE_KEY_BASE64="your-private-key"
GITHUB_APP_NAME="Classmoji-xxxxxx"
GITHUB_CLIENT_ID="your-client-id"
GITHUB_CLIENT_SECRET="your-client-secret"
```

Sign in with your GitHub account. You'll be automatically added to the dev classroom as Owner, TA, and Student.

### What you can work on with this setup

- UI changes — components, styles, dark mode, layouts
- Database schema changes and migrations
- Student, instructor, and TA views and workflows
- Grading, tokens, calendar, and modules
- Slide editor
- Quiz UI and grading logic
- Anything that doesn't require real GitHub repositories or live webhooks

## 5. Full setup

Needed when working on anything GitHub-integrated: repository creation, assignment auto-setup, feedback PRs, pages (stored on GitHub), webhook event handling, or background jobs in `packages/tasks`. These all go together.

### Create a test GitHub organization

Create a free GitHub organization to use for testing. Then go to your GitHub App settings and install the app on that org.

### Start the webhook tunnel

```bash
npm run hook:github
```

This prints a public tunnel URL. Set it as the webhook URL in your GitHub App settings, then add the secret to your `.env`:

```env
GITHUB_WEBHOOK_SECRET=your-webhook-secret
```

### Start Trigger.dev

```bash
npm run trigger:dev
```

This uses your local `.env` directly — no Infisical needed.
